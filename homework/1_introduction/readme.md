# Задача 2

Выберите один из вариантов платформы в зависимости от задачи. Здесь нет однозначно верного ответа так как все зависит от конкретных условий: финансирование, компетенции специалистов, удобство использования, надежность, требования ИБ и законодательства, фазы луны.

Тип платформы:

физические сервера;
паравиртуализация;
виртуализация уровня ОС;
Задачи:

высоконагруженная база данных MySql, критичная к отказу;
различные web-приложения;
Windows-системы для использования бухгалтерским отделом;
системы, выполняющие высокопроизводительные расчёты на GPU.
Объясните критерии выбора платформы в каждом случае.



## 1️. Высоконагруженная база данных MySQL, критичная к отказу

**Рекомендуемая платформа:**  
**Физические серверы (bare metal)**

**Обоснование:**
- **Производительность:** минимальные накладные расходы, высокая I/O-производительность.
- **Предсказуемость:** отсутствие прослоек гипервизора или контейнеров.
- **Надёжность:** проще реализовать кластеризацию (MySQL Group Replication, Percona XtraDB Cluster).
- **Информационная безопасность:** физическая изоляция соответствует строгим требованиям.

**Альтернатива:** паравиртуализация с выделенными ресурсами (например, VMware vSphere, KVM), если требуется гибкость и миграция без простоев.

---

## 2️. Различные web-приложения

**Рекомендуемая платформа:**  
**Виртуализация уровня ОС (контейнеризация — Docker, Kubernetes, OpenVZ)**

**Обоснование:**
- **Масштабируемость:** простое горизонтальное масштабирование.
- **Удобство развёртывания:** поддержка CI/CD и DevOps-практик.
- **Экономия ресурсов:** общее ядро ОС, без лишней виртуализации.
- **Изоляция:** достаточно для большинства веб-сервисов.

**Альтернатива:** паравиртуализация — при необходимости изоляции разных заказчиков или окружений.

---

## 3️. Windows-системы для бухгалтерского отдела

**Рекомендуемая платформа:**  
**Паравиртуализация (Hyper-V, VMware, KVM)**

**Обоснование:**
- **Совместимость:** поддержка полноценных Windows-гостевых систем.
- **Управляемость:** централизованное администрирование, резервное копирование.
- **Безопасность:** изоляция от других подсистем.
- **Гибкость:** снапшоты, клонирование, быстрые откаты.

---

## 4️. Системы для высокопроизводительных расчётов на GPU

**Рекомендуемая платформа:**  
**Физические серверы (bare metal)**

**Обоснование:**
- **Доступ к GPU:** отсутствие виртуализационных ограничений.
- **Максимальная производительность:** важна при ML/HPC-задачах.
- **Простота настройки CUDA и драйверов.**
- **Минимальные задержки.**

**Альтернатива:** паравиртуализация с GPU passthrough (например, NVIDIA vGPU) — если нужно делить ресурсы между задачами.

# Задача 3

Выберите подходящую систему управления виртуализацией для предложенного сценария. Опишите ваш выбор.

Сценарии:

1. 100 виртуальных машин на базе Linux и Windows, общие задачи, нет особых требований. Преимущественно Windows based-инфраструктура, требуется реализация программных балансировщиков нагрузки, репликации данных и автоматизированного механизма создания резервных копий.
2. Требуется наиболее производительное бесплатное open source-решение для виртуализации небольшой (20-30 серверов) инфраструктуры на базе Linux и Windows виртуальных машин.
3. Необходимо бесплатное, максимально совместимое и производительное решение для виртуализации Windows-инфраструктуры.
4. Необходимо рабочее окружение для тестирования программного продукта на нескольких дистрибутивах Linux.

## 1. 100 ВМ, Windows-инфраструктура, балансировка, репликация, бэкапы

**Рекомендуемая платформа:** : VMware vSphere Suite (vCenter Server + ESXi)

**Обоснование:**
- **Оптимальная совместимость с Windows** - многолетняя отработанная интеграция с ОС Windows
- **Масштабируемость** - идеально подходит для управления 100+ ВМ через vCenter Server
- **Корпоративные функции**:
  - **Балансировка нагрузки**: vSphere DRS (Distributed Resource Scheduler)
  - **Репликация данных**: встроенная vSphere Replication
  - **Резервное копирование**: поддержка VADP API для интеграции с Veeam, Commvault

**Ключевые преимущества:**
- Единая панель управления через vCenter
- Зрелая экосистема инструментов
- Простота автоматизации задач

**Альтернатива:** 
- Microsoft Hyper-V + SCVMM (платный, сложнее в настройке)
- Proxmox VE (менее отполированные инструменты для Windows)

---

## 2. Производительное бесплатное Open Source решение для 20-30 серверов

**Рекомендуемая платформа:** : Proxmox Virtual Environment (VE)

**Обоснование:**
- **Высокая производительность** - тип 1 гипервизор на базе KVM
- **Полная бесплатность** - функциональная community-версия
- **Open Source** - открытый исходный код
- **Удобство управления** - интуитивный веб-интерфейс

**Ключевые преимущества:**
- Встроенные функции кластеризации
- Live migration без простоя
- Поддержка контейнеров LXC и виртуальных машин
- Интегрированные бэкапы и репликация

**Альтернатива:** 
- XCP-ng (меньше сообщество и документация)
- oVirt (избыточен для небольшой инфраструктуры)

---

## 3. Бесплатное и совместимое решение для Windows-инфраструктуры

**Рекомендуемая платформа:** : Microsoft Hyper-V Server (бесплатная версия)

**Обоснование:**
- **Максимальная совместимость** - родная платформа от Microsoft
- **Бесплатность** - бесплатный автономный продукт
- **Производительность** - тип 1 гипервизор

### Особенности:
- Управление через PowerShell или удаленный Диспетчер Hyper-V
- Идеален для "голой" производительной платформы
- Гарантированная поддержка всех версий Windows

**Альтернатива:** 
- Proxmox VE (требует настройки VirtIO драйверов)
- KVM/QEMU (сложнее в настройке для Windows)

---

## 4. Тестовое окружение для Linux дистрибутивов

**Рекомендуемая платформа:** : Oracle VM VirtualBox**

**Обоснование:**
- **Простота использования** - тип 2 гипервизор, устанавливается как приложение
- **Удобство тестирования** - снимки состояний (snapshots), клонирование
- **Отличная поддержка Linux** - гостевые дополнения для большинства дистрибутивов
- **Кроссплатформенность** - работает на Windows, macOS, Linux

**Ключевые преимущества:**
- Быстрое развертывание тестовых сред
- Общие папки и буфер обмена
- Легкий перенос конфигураций ВМ

**Альтернатива:** 
- VMware Workstation Pro/Fusion (платный)
- QEMU/KVM (требует больше ручной настройки)

# Задача 4

Опишите возможные проблемы и недостатки гетерогенной среды виртуализации (использования нескольких систем управления виртуализацией одновременно) и что необходимо сделать для минимизации этих рисков и проблем. Если бы у вас был выбор, создавали бы вы гетерогенную среду или нет?

## Возможные проблемы и недостатки

### 1. Сложность управления и администрирования
- **Разные интерфейсы управления** - необходимость знания vCenter, SCVMM, Proxmox GUI и других консолей
- **Разная терминология и концепции** - различные реализации кластеризации, миграции, резервного копирования
- **Отсутствие единой точки управления** - невозможность централизованного мониторинга и управления без дополнительных инструментов

### 2. Высокие операционные расходы (OPEX)
- **Обучение персонала** - инвестиции в обучение администраторов разным платформам
- **Сложное лицензирование** - различные модели лицензирования (per CPU, per core, подписки)
- **Низкая операционная эффективность** - разные процедуры для рутинных задач на каждой платформе

### 3. Проблемы безопасности
- **Увеличенная поверхность атаки** - больше сервисов, портов и уязвимостей для отслеживания
- **Сложность унификации политик безопасности** - разные механизмы реализации политик
- **Риск неконсистентности настроек** - возможность упустить важные настройки из-за различий платформ

### 4. Сложности с резервным копированием и DR
- **Разные API и механизмы снимков** - необходимость нескольких модулей в инструментах резервного копирования
- **Отсутствие единой стратегии аварийного восстановления** - разные процедуры восстановления для разных платформ
- **Увеличение RTO (Recovery Time Objective)** - сложность координации восстановления

### 5. Проблемы производительности и использования ресурсов
- **"Силосы" ресурсов** - невозможность перераспределения ресурсов между разными гипервизорами
- **Разные инструменты мониторинга** - сложность комплексного анализа производительности
- **Неоптимальное использование оборудования** - невозможность балансировки нагрузки между платформами

### 6. Ограничения мобильности рабочих нагрузок
- **Отсутствие кросс-платформенной живой миграции** - требуется остановка ВМ для переноса между гипервизорами
- **Сложность конвертации форматов ВМ** - необходимость дополнительных инструментов конвертации

## Заключение

В большинстве случаев стандартизация на одной платформе обеспечивает лучшую операционную эффективность и снижает совокупную стоимость владения, поэтому если бы у меня был выбор, я бы не создавал гетерогенную среду, а создавал гомогенную
